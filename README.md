# 마이 스케줄 (My Schedule)

개인용 데일리 허브 PWA. 일정은 구글 캘린더에 맡기고, 이 앱은 **투두 · 루틴 · 웹소설 작업 기록 · 관람/독서 로그**를 하루 단위로 종합해서 보여주는 데 집중합니다.

- **배포 주소:** https://yewonshin-kr.github.io/my-schedule/
- **형태:** 프레임워크 없는 단일 HTML 파일(`index.html`) 바닐라 JS. 빌드 과정이 없어 파일을 고치고 그대로 배포하면 끝입니다.
- **사용 방식:** iPhone Safari에서 열고 **공유 → 홈 화면에 추가**하면 앱처럼 아이콘으로 실행됩니다(PWA, 오프라인 지원).

## 화면 구성

하단 탭 5개로 구성되어 있습니다.

### 🏠 Home (캘린더)

- **주간뷰가 기본.** 요일별로 그날의 투두 · 루틴 · 웹소설 업로드/작업기록 · 로그를 색상 칩으로 한 화면에서 볼 수 있습니다. 좌우 스와이프 또는 미니 캘린더 탭으로 주 이동, 상단 토글로 월간뷰 전환도 가능합니다.
- **투두 중심 등록.** ＋ 버튼으로 항목을 만들면 기본이 "투두"(체크 가능한 일회성 항목)이고, 정기·기간 유형도 선택할 수 있습니다. 제목만 입력하고 엔터를 쳐도 바로 저장됩니다.
- **Schedule 필터**는 한국/일본 공휴일과 구글 캘린더 읽기 연동을 묶은 그룹입니다. 구글 드라이브 연동을 켜면 구글 캘린더 일정이 읽기 전용으로 함께 표시됩니다.
- 날짜를 탭하면 그날의 모든 항목(투두 체크, 루틴, 작업 글자 수, 로그 등)을 모아 보여주는 종합 시트가 열립니다. 지난 미완료 투두는 오늘 시트에 "N일 지남"으로 이월 표시됩니다.

### ✅ Routine

- 고정 요일 / N일 간격 / 주 N회(flex) 3가지 반복 방식을 지원합니다.
- 루틴마다 **종료일**을 설정할 수 있고(기본값 없이 무기한), 종료일이 지나면 캘린더·통계 어디에도 표시되지 않습니다.
- 달성률은 **해당 월 전체 기준**(과거+미래 예정 포함)으로 계산됩니다. 예: 월수금 루틴이면 그 달의 월수금 전체 횟수가 분모입니다.
- 상단 **[목록 | Report]** 토글로 리포트 화면을 볼 수 있습니다 — 이달 평균 달성률, 루틴별 달성률 바, 최근 6개월 추이, 100% 달성 루틴 표시.

### 📖 Novel (웹소설)

- 작품별 연재 관리: 업로드 요일/주기, 회차 예약, 출간 일정, 교정 일정, 작업 기록(글자 수).
- 회차·출간 목록이 많아지면 상세 화면에 **전체 관리(페이징)** 화면이 열립니다(20건씩, 최신순).
- 작업 기록은 수기 등록 또는 **CSV/구글 시트 가져오기**로 한꺼번에 입력할 수 있습니다(작업 기록 등록 모달 안에 통합되어 있음).
- 상단 **[목록 | Report]** 토글: 이달 글자수·작업일·일평균, 최근 6개월 추이, 작품별/작업종류별 글자수, 올해 누적.
- 좌우 스와이프로 월 이동이 가능합니다.

### ⭐ Log (구 Review)

- 영화 · 드라마 · 애니메이션 · 게임 · 책 · 연극/뮤지컬/콘서트 등 카테고리별 관람·독서 기록.
- 상단 구성 순서: **[Calendar · List · Album · Report] 뷰 전환 → 카테고리 선택 → (뷰에 따라) 월 이동 또는 정렬·필터**.
- 캘린더뷰에서 빈 날짜를 탭하면 그 날짜로 바로 기록을 등록할 수 있습니다.
- **기록만 남기기** 체크박스: 날짜를 기억 못 하는 옛날 기록을 아카이브할 때 사용합니다. 켜면 시작일이 시스템상 `1000-01-01`로 저장되고, 캘린더에는 안 뜨고 List/Album/Report에서만 "기록"으로 표시됩니다.
- **정보 검색**: 제목 입력 후 "정보 검색"을 누르면 카테고리에 맞는 외부 DB에서 검색해 포스터/표지와 **줄거리·소개**까지 함께 가져옵니다. 가져온 정보는 상세 화면 맨 아래에서 직접 수정하거나 전체 삭제할 수 있습니다.
- **Report 뷰**: 연도 선택 칩, 기록 수/평균 별점/N차 관람 요약, 연도별·월별 추이, 카테고리 분포, 별점 분포, 다시 본 작품 TOP 5, 본 곳(극장/OTT) 분포.

### ⚙️ Settings

- 구글 캘린더 가져오기(.ics), 데이터 백업/복원(JSON), 스케줄 태그·로그 카테고리 등 선택지 관리, 검색 연동(중계 서버) 안내.

## 데이터 저장 방식

- 모든 데이터는 브라우저 **localStorage**의 `psm_v1` 키 하나에 JSON으로 저장됩니다. 기기를 바꾸면 데이터가 옮겨가지 않으므로 **설정 → 데이터 백업**을 주기적으로 사용하세요.
- **구글 드라이브 동기화**가 구현되어 있어(`drive.file` 스코프), 켜두면 앱 데이터가 드라이브에도 자동 저장/복원됩니다.
- 주요 필드: `schedules`(투두/정기/기간), `routines`, `reviews`(Log), `works`/`worklogs`(웹소설), `filters`, `ui`(뷰 상태) 등. 스키마가 바뀔 때마다 `load()` 함수 안에서 옛 데이터를 자동 마이그레이션하도록 만들어져 있습니다.

## 정보 검색 연동 (중계 서버)

Log 탭의 "정보 검색"은 앱이 외부 API를 직접 호출하지 않고, **Cloudflare Worker 중계 서버**를 거칩니다. 브라우저에서 막히는 CORS 문제를 피하고, API 키를 앱 코드에 노출하지 않기 위해서입니다.

| 카테고리 | 소스 | 제공 정보 |
|---|---|---|
| 책 | 알라딘 | 표지, 저자, 책 소개 |
| 영화·드라마·애니메이션 | TMDB | 포스터, 원제, 줄거리 |
| 게임 | RAWG | 스크린샷, 플랫폼, 소개 |
| 연극/뮤지컬/콘서트/전시 | 한국문화정보원(KOPIS 계열) | 썸네일, 장소·기간, 소개 |

중계 서버 코드는 `worker.js`입니다. **API 키는 코드에 적지 않고 Cloudflare 대시보드의 Secret으로 등록**합니다:

1. themoviedb.org, rawg.io, 알라딘 TTB, 문화정보원(data.go.kr)에서 각각 무료 키를 발급받습니다.
2. Cloudflare 대시보드 → Workers & Pages → 해당 워커 → **Settings → Variables and Secrets → Add → Type: Secret** 으로 아래 4개를 등록합니다.

   | Name | 값 |
   |---|---|
   | `ALADIN_KEY` | 알라딘 TTB 키 |
   | `CULTURE_KEY` | 문화정보원 serviceKey |
   | `TMDB_KEY` | TMDB v3 API 키 |
   | `RAWG_KEY` | RAWG API 키 |

3. `worker.js` 내용을 워커에 붙여넣고 Deploy 합니다.

키가 없는 카테고리는 에러 대신 "연동을 추가한 뒤 사용할 수 있어요" 안내만 뜨고 나머지 기능은 정상 동작합니다.

## 파일 구조

```
my-schedule/
├─ index.html              # 앱 전체 (HTML+CSS+JS 단일 파일)
├─ manifest.webmanifest     # PWA 매니페스트
├─ sw.js                    # 서비스 워커 (오프라인 캐시, 네트워크 우선 전략)
├─ icon-192.png / icon-512.png / icon-maskable-512.png
└─ worker.js                # (별도 배포) 정보 검색 중계 서버 — Cloudflare Worker
```

`worker.js`는 이 저장소가 아니라 **Cloudflare Workers 대시보드에 별도로 배포**하는 코드입니다. GitHub Pages는 정적 파일만 서빙하므로, 서버가 필요한 부분(API 키를 숨기고 외부 API를 대신 호출하는 역할)만 워커로 분리되어 있습니다.

## 로컬에서 수정하기

빌드 도구가 없으므로 `index.html`을 텍스트 에디터로 직접 수정하고, 브라우저로 파일을 열거나 GitHub Pages에 푸시해 확인하면 됩니다.

```bash
git clone https://github.com/yewonshin-kr/my-schedule.git
cd my-schedule
# index.html을 수정한 뒤
open index.html   # 또는 file:// 로 직접 열어서 확인 (일부 기능은 http(s) 환경에서만 동작할 수 있음)
```

수정 후 커밋·푸시하면 GitHub Pages가 자동으로 반영합니다.

## 배포 방법

### 1. 앱 (GitHub Pages)

```bash
git add index.html
git commit -m "설명"
git push
```
잠시 후 https://yewonshin-kr.github.io/my-schedule/ 에 반영됩니다. PWA 서비스 워커가 네트워크 우선 전략이라, 온라인 상태에서 한 번 열면 최신 버전이 적용됩니다.

### 2. 중계 서버 (Cloudflare Workers)

`worker.js`를 대시보드에서 붙여넣기 하거나 `wrangler`로 배포합니다.

```bash
# wrangler 사용 시
wrangler deploy worker.js
```

배포 후 [정보 검색 연동](#정보-검색-연동-중계-서버) 절차대로 4개 Secret이 등록되어 있는지 확인하세요.

## 알려진 제약

- 데이터가 브라우저 로컬에만 저장되므로, 기기를 바꾸기 전엔 반드시 백업하거나 구글 드라이브 동기화를 켜두어야 합니다.
- 삼성 캘린더는 공개 API가 없어 직접 연동이 불가능합니다. 삼성 캘린더를 구글 계정과 동기화해두면 구글 캘린더 연동으로 우회해서 볼 수 있습니다.
- 정보 검색은 Cloudflare Worker가 정상 배포되어 있고 해당 카테고리의 Secret이 등록되어 있어야 동작합니다.
