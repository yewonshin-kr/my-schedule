# Daily Hub (마이 스케줄)

개인용 데일리 허브 PWA. 일정은 구글 캘린더에 맡기고, 이 앱은 **투두 · 루틴 · 웹소설 작업 기록 · 관람/독서 로그**를 하루 단위로 종합해서 보여주는 데 집중합니다.

- **배포 주소:** https://yewonshin-kr.github.io/my-schedule/
- **형태:** 프레임워크 없는 단일 HTML 파일(`index.html`) 바닐라 JS. 빌드 과정이 없어 파일을 고치고 그대로 배포하면 끝입니다.
- **설치:** 안드로이드 Chrome에서 열고 메뉴 → **앱 설치** (또는 홈 화면에 추가). iOS Safari는 공유 → 홈 화면에 추가. PWA라 오프라인에서도 실행됩니다.
- 안드로이드 설치형에서는 **예매 앱 공유 → Daily Hub 선택 시 일정 자동 등록**(제목·날짜·시간 파싱), 아이콘 롱프레스 바로가기(작업 기록 / 일정 추가)도 동작합니다.

## 화면 구성

하단 탭 5개로 구성되어 있습니다.

### 🏠 Home (캘린더)

- **주간뷰가 기본.** 요일별로 그날의 투두 · 루틴 · 웹소설 업로드/작업기록 · 로그를 색상 칩으로 한 화면에서 볼 수 있습니다. 좌우 스와이프 또는 미니 캘린더 탭으로 주 이동, 상단 토글로 월간뷰 전환도 가능합니다.
- **투두 중심 등록.** ＋ 버튼으로 항목을 만들면 기본이 **일회성**(체크 가능한 투두)이고, **정기**(주/일/월 단위 반복 — 격주·매월 말일 등 지원) · **기간**(시작일~종료일) 유형도 선택할 수 있습니다. 제목만 입력하고 엔터를 쳐도 바로 저장됩니다.
- **Schedule 필터 그룹**: 한국 공휴일 · 일본 공휴일(각각 토글, 2025~2027 내장 데이터) · 구글 캘린더 읽기 연동을 묶어서 관리합니다. 태그(예: LOL)별 표시/숨김도 상세필터에서 조절합니다.
- 날짜를 탭하면 그날의 모든 항목(투두 체크, 루틴 체크, 작업 글자 수, 로그 등)을 모아 보여주는 종합 시트가 열립니다. 지난 미완료 투두는 오늘 시트에 "N일 지남"으로 이월 표시됩니다.
- 안드로이드 뒤로가기는 시트 닫기 → 상세 닫기 → Home 복귀 순서로 동작하고, 새로고침해도 보던 탭이 유지됩니다.

### ✅ Routine

- 고정 요일(격주 등 주 간격 포함) / N일 간격 / 매월 특정일·말일 / 주 N회(flex) 반복 방식을 지원합니다.
- 루틴마다 **종료일**을 설정할 수 있고(기본 무기한), 종료일이 지나면 캘린더·통계 어디에도 표시되지 않습니다.
- 달성률은 **해당 월 전체 기준**(과거+미래 예정 포함)으로 계산됩니다. 예: 월수금 루틴이면 그 달의 월수금 전체 횟수가 분모입니다. 목표일이 아닌 날 체크는 보충 기록으로 남습니다.
- 상단 **[목록 | Report]** 토글 — 이달 평균 달성률, 루틴별 달성률 바, 최근 6개월 추이, 100% 달성 루틴 표시.

### 📖 Novel (웹소설)

- 작품별 연재 관리: 회차 예약(기간+주기 방식 — 매일/격일/N일/요일 지정, 미리보기에서 날짜 개별 수정), 출간 일정, 교정 일정, 작업 기록(집필/시놉시스/교정/기타 + 글자 수).
- 회차·출간 목록이 많아지면 상세 화면에서 **전체 관리(20건씩 페이징, 최신순)** 화면이 열립니다.
- 작업 기록은 수기 등록 또는 **CSV/구글 시트 가져오기**(작업 기록 모달에 통합, 중복 자동 스킵)로 일괄 입력할 수 있습니다.
- **월별 목표**: 주 N회·월 글자수 목표를 달마다 다르게 설정할 수 있고, 설정하지 않은 달은 직전 설정을 따라갑니다.
- 상단 **[목록 | Report]** 토글 — 이달 글자수·작업일·일평균, 최근 6개월 추이, 작품별/작업종류별 글자수, 올해 누적.
- 좌우 스와이프로 월 이동이 가능합니다.

### ⭐ Log (관람·독서 기록)

- 영화 · 드라마 · 애니메이션 · 게임 · 책 · 연뮤/콘서트/전시 등 카테고리별 기록. 카테고리는 자유롭게 추가/삭제할 수 있습니다.
- **카테고리별 입력 필드 설정**: 카테고리마다 날짜 방식(시작일만/기간/시작일+시간), 본 곳 유형(영화관/OTT/자유 입력/없음), 상영관·좌석·메모·블로그 리뷰 링크 필드를 켜고 끌 수 있습니다. (예: 영화=시작일+영화관+상영관+좌석, 드라마=기간+OTT, 책·게임=기간+메모)
- 상단 구성: **[Calendar · List · Album · Report] 뷰 전환 → 카테고리 선택 → (뷰에 따라) 월 이동 또는 정렬·필터**. 캘린더뷰는 기간 기록을 기간 전체에 표시하고, 빈 날짜를 탭하면 그 날짜로 바로 등록할 수 있습니다.
- **N차 기록(그룹)**: 같은 작품의 기록들을 그룹으로 연결해 1차·2차…로 관리합니다. 새 기록을 "기존 기록에 회차로 추가"하거나 언제든 연결/해제할 수 있습니다.
- **기록만 남기기**: 날짜를 기억 못 하는 옛 기록은 날짜 없이 저장됩니다(캘린더 제외, List/Album/Report에만 표시). 구버전의 `1000-01-01` 데이터는 자동 변환됩니다.
- **정보 검색**: 제목 입력 후 "정보 검색" → 카테고리에 맞는 외부 DB에서 포스터/표지·날짜·**줄거리/소개**까지 가져옵니다. "더 보기" 페이징 지원. 가져온 정보는 상세에서 수정·삭제 가능합니다.
- **Report 뷰**: 연도 선택, 기록 수/평균 별점/N차 관람 요약, 연도별·월별 추이, 카테고리·별점 분포, 다시 본 작품 TOP 5, 본 곳(극장/OTT) 분포.

### ⚙️ Settings

- 구글 드라이브 동기화(연결/해제/지금 동기화), 구글 캘린더 가져오기(.ics), 데이터 백업/복원(JSON), 데이터 리스트 관리(스케줄 태그 · 로그 카테고리와 필드 설정 · 본 곳/OTT 목록 · 루틴 그룹), 검색 연동(중계 서버) 안내.

## 데이터 저장 방식

- 모든 데이터는 브라우저 **localStorage**의 `psm_v1` 키 하나에 JSON으로 저장됩니다.
- **구글 드라이브 동기화**(`drive.file` 스코프)를 켜두면 드라이브의 `my-schedule-data.json`에 자동 저장/복원되어 여러 기기에서 같은 데이터를 씁니다(최신 `updatedAt` 우선). 액세스 토큰을 로컬에 캐시(약 1시간)하고 만료 시 첫 터치에서 무음 재발급해 재로그인을 최소화합니다.
- 주요 필드: `schedules`(일회성 투두/정기/기간), `routines`, `reviews`(Log), `works`/`worklogs`(웹소설), `novelGoals`(월별 목표), `catConfig`(카테고리 필드 설정), `filters`, `ui`(뷰 상태) 등. 스키마가 바뀔 때마다 `load()` 안에서 옛 데이터를 자동 마이그레이션합니다.

## 정보 검색 연동 (중계 서버)

Log 탭의 "정보 검색"은 앱이 외부 API를 직접 호출하지 않고 **Cloudflare Worker 중계 서버**를 거칩니다. 브라우저에서 막히는 CORS 문제를 피하고, API 키를 앱 코드에 노출하지 않기 위해서입니다.

카테고리 이름으로 소스를 자동 선택합니다:

| 카테고리(이름에 포함된 키워드) | 엔드포인트 | 소스 | 제공 정보 |
|---|---|---|---|
| 책 · 도서 · 소설 | `/book` | 알라딘 | 표지, 저자, 책 소개 |
| 게임 | `/game` | RAWG | 스크린샷, 플랫폼, 소개 |
| 연뮤 · 공연 · 콘서트 · 전시 · 뮤지컬 · 연극 · 페스티벌 | `/perf` | 한국문화정보원(한눈에보는문화정보) | 썸네일, 장소·기간, 소개 |
| 그 외 전부 (영화·드라마·애니메이션 등) | `/tmdb` | TMDB | 포스터, 원제, 줄거리 |

응답은 공통 스키마 `{ title, img, date, sub, info }`이며 페이지 파라미터 `p`와 `hasMore`(TMDB/RAWG)를 지원합니다. `/perfraw`는 문화정보원 원본 XML 확인용 디버그 엔드포인트입니다.

중계 서버 코드는 `worker.js`입니다. **API 키는 코드에 적지 않고 Cloudflare Secret으로 등록**합니다:

1. 알라딘 TTB, 문화정보원(data.go.kr), themoviedb.org, rawg.io에서 각각 무료 키를 발급받습니다.
2. Cloudflare 대시보드 → Workers & Pages → 해당 워커(`my-schedule-api`) → **Settings → Variables and Secrets → Add → Type: Secret** 으로 등록:

   | Name | 값 |
   |---|---|
   | `ALADIN_KEY` | 알라딘 TTB 키 |
   | `CULTURE_KEY` | 문화정보원 serviceKey |
   | `TMDB_KEY` | TMDB v3 API 키 |
   | `RAWG_KEY` | RAWG API 키 |

3. `worker.js` 내용을 워커에 붙여넣고 Deploy 합니다.

키가 없는 카테고리는 에러 대신 빈 결과/안내만 뜨고 나머지 기능은 정상 동작합니다. 워커 응답에는 1시간 캐시가 걸려 있어, 방금 키를 등록했는데 빈 결과가 나오면 검색어를 바꿔 확인하세요.

## 구글 연동

- GCP 프로젝트의 OAuth 웹 클라이언트를 사용합니다(승인된 자바스크립트 원본: `https://yewonshin-kr.github.io`).
- 스코프: `drive.file`(앱 데이터 파일) · `calendar.readonly`(캘린더 읽기 표시) · `spreadsheets.readonly`(작업 기록 시트 가져오기).
- 테스트 모드 앱이라 최초 로그인 시 "확인되지 않은 앱" 경고가 뜨는 것이 정상입니다.

## 파일 구조

```
my-schedule/
├─ index.html              # 앱 전체 (HTML+CSS+JS 단일 파일)
├─ manifest.webmanifest     # PWA 매니페스트 (Daily Hub, 공유 대상·바로가기 포함)
├─ sw.js                    # 서비스 워커 (오프라인 캐시, 네트워크 우선 전략)
├─ icon-192.png / icon-512.png / icon-maskable-512.png
└─ worker.js                # (별도 배포) 정보 검색 중계 서버 — Cloudflare Worker
```

`worker.js`는 이 저장소가 아니라 **Cloudflare Workers 대시보드에 별도로 배포**하는 코드입니다. GitHub Pages는 정적 파일만 서빙하므로, 서버가 필요한 부분(API 키를 숨기고 외부 API를 대신 호출)만 워커로 분리되어 있습니다.

## 로컬에서 수정하기

빌드 도구가 없으므로 `index.html`을 텍스트 에디터로 직접 수정하고, 브라우저로 열거나 GitHub Pages에 푸시해 확인하면 됩니다.

```bash
git clone https://github.com/yewonshin-kr/my-schedule.git
cd my-schedule
# index.html을 수정한 뒤
open index.html   # file:// 로 직접 확인 가능 (구글 연동·공유 등록 등 일부 기능은 https 환경에서만 동작)
```

## 배포 방법

### 1. 앱 (GitHub Pages)

```bash
git add index.html
git commit -m "설명"
git push
```
잠시 후 https://yewonshin-kr.github.io/my-schedule/ 에 반영됩니다. 서비스 워커가 네트워크 우선 전략이라, 온라인 상태에서 앱을 다시 열면 최신 버전이 적용됩니다. (웹 UI에서 파일 덮어쓰기 업로드로도 동일하게 배포됩니다.)

### 2. 중계 서버 (Cloudflare Workers)

`worker.js`를 대시보드에서 붙여넣기 하거나 `wrangler`로 배포합니다.

```bash
wrangler deploy worker.js
```

배포 후 [정보 검색 연동](#정보-검색-연동-중계-서버) 절차대로 4개 Secret이 등록되어 있는지 확인하세요.

## 알려진 제약

- 데이터가 브라우저 로컬에 저장되므로, 드라이브 동기화를 켜두지 않았다면 기기를 바꾸기 전에 반드시 백업하세요.
- 안드로이드 홈 화면 위젯·푸시 알림은 PWA 플랫폼 한계로 지원하지 않습니다.
- 삼성 캘린더는 공개 API가 없어 직접 연동이 불가능합니다. 삼성 캘린더를 구글 계정과 동기화해두면 구글 캘린더 연동으로 우회해서 볼 수 있습니다.
- 내장 공휴일 데이터는 2025~2027년입니다. 2028년이 가까워지면 갱신이 필요합니다.
- 정보 검색은 Cloudflare Worker가 배포되어 있고 해당 카테고리의 Secret이 등록되어 있어야 동작합니다.
